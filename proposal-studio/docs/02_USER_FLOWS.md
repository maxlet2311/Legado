# Objetivo

Este documento describe el recorrido funcional que realiza un Productor Asesor de Seguros para construir una propuesta comercial dentro de Proposal Studio.

El flujo fue diseñado para acompañar un proceso consultivo y no un proceso de carga de datos.

Cada fase tiene un objetivo específico y genera información que será utilizada por las fases posteriores hasta construir la propuesta final.

---

# Principios del flujo

Todo flujo dentro de Proposal Studio debe respetar los siguientes principios:

- La estrategia siempre precede al producto.
- El usuario debe sentirse guiado, nunca perdido.
- Cada fase debe tener un único objetivo.
- El sistema debe reducir decisiones innecesarias.
- La propuesta debe construirse progresivamente.
- El usuario puede volver y modificar cualquier fase antes de exportar.

# Fase 1 — Descubrimiento

## Objetivo

Comprender qué quiere lograr el cliente antes de comenzar a construir la propuesta.

El producto nunca debe comenzar preguntando qué seguro desea vender el asesor.

Debe comenzar comprendiendo la necesidad del cliente.

## Opciones

- Proteger a su familia
- Construir ahorro
- Planificar su retiro
- Proteger su empresa
- Proteger a los socios
- Fidelizar colaboradores
- Estrategia personalizada

## Acciones del sistema

Al seleccionar un objetivo, Proposal Studio configura automáticamente:

- narrativa sugerida;
- categorías de beneficios;
- estructura inicial de la propuesta;
- plantillas recomendadas;
- bloques sugeridos;
- CTA sugeridos;
- configuración visual inicial.

Todas estas configuraciones pueden modificarse posteriormente.

## Resultado esperado

El sistema identifica el objetivo principal del cliente y prepara automáticamente el resto del flujo.

---

# Fase 2 — Contexto del cliente

## Objetivo

Recopilar la información necesaria para personalizar la propuesta.

## Información principal

- tipo de cliente;
- datos personales o empresariales;
- contexto general;
- objetivo declarado;
- plazo;
- moneda;
- asesor responsable.

## Acciones del sistema

Según el tipo de cliente seleccionado, Proposal Studio adapta automáticamente:

- terminología;
- beneficios sugeridos;
- estructura narrativa;
- notas legales;
- componentes disponibles.

## Resultado esperado

La propuesta queda contextualizada para el perfil del cliente.

---

# Fase 3 — Diagnóstico

## Objetivo

Comprender la situación actual del cliente e identificar las razones que justifican la estrategia propuesta.

## Información analizada

- situación actual;
- riesgos;
- oportunidades;
- prioridades;
- objetivos.

## Ejemplos

- falta de protección familiar;
- ausencia de ahorro sistemático;
- necesidad de retiro complementario;
- dependencia económica;
- riesgo societario;
- necesidad de liquidez;
- retención de talento.

## Acciones del sistema

La información recopilada será utilizada para construir automáticamente el Resumen Ejecutivo de la propuesta.

## Resultado esperado

La plataforma dispone de los argumentos necesarios para justificar la recomendación.

---

# Fase 4 — Diseño de la estrategia

Esta fase representa el núcleo de Proposal Studio.

Aquí el asesor construye la solución que será presentada al cliente.

La estrategia siempre precede a la comparación de alternativas.

---

## 4.1 Alternativas

El asesor puede crear hasta seis alternativas comerciales.

Cada alternativa puede clasificarse como:

- Protección
- Ahorro
- Vida con ahorro
- Retiro
- Empresa

El usuario puede:

- crear;
- editar;
- ocultar;
- ordenar;
- destacar una alternativa como recomendada.

---

## 4.2 Beneficios

Proposal Studio sugiere automáticamente beneficios relacionados con el objetivo del cliente.

El asesor puede:

- activar;
- desactivar;
- editar;
- crear nuevos;
- guardar en la biblioteca.

---

## 4.3 Estrategia recomendada

El asesor construye la narrativa principal de la propuesta.

Puede hacerlo:

- escribiendo manualmente;
- utilizando una plantilla;
- combinando ambas opciones.

La estrategia debe responder:

- qué entendimos;
- por qué existe una necesidad;
- cuál es la solución recomendada;
- qué resultado se espera obtener.

## Resultado esperado

La propuesta queda completamente estructurada.

---

# Fase 5 — Comparativa

## Objetivo

Permitir que el cliente compare las alternativas de manera clara y sencilla.

## Acciones del usuario

Seleccionar la información que será visible en la tabla comparativa.

## Acciones del sistema

La tabla se genera automáticamente utilizando únicamente las alternativas visibles.

## Resultado esperado

El cliente puede comprender fácilmente las diferencias entre cada alternativa.

---

# Fase 6 — Próximos pasos

## Objetivo

Cerrar la propuesta indicando cuál es la recomendación final y cómo avanzar.

## Acciones del usuario

Editar:

- título;
- mensaje final;
- botón principal;
- mensaje de WhatsApp;
- alternativa recomendada.

## Resultado esperado

La propuesta concluye con una invitación clara para continuar el proceso comercial.

---

# Vista previa

## Objetivo

Permitir revisar la propuesta completa antes de exportarla.

La vista previa debe representar exactamente el documento PDF final.

El usuario puede regresar a cualquier fase para realizar modificaciones.

---

# Exportación

## Formatos soportados

- A4 Vertical
- A4 Horizontal

## Opciones

- mostrar u ocultar secciones;
- incluir nota legal;
- incluir footer;
- exportar PDF.

---

# Estados del flujo

Toda propuesta atraviesa los siguientes estados:

- Nueva
- En edición
- Lista para revisión
- Lista para exportar
- Exportada
- Archivada

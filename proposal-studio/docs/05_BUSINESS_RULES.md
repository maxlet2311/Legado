# Objetivo

Este documento define todas las reglas de negocio de Proposal Studio™.

Toda decisión automática tomada por el sistema deberá estar respaldada por una regla documentada aquí.

El objetivo es desacoplar la lógica comercial del código fuente.

Las pantallas muestran información.

El modelo de datos la almacena.

Este documento explica cómo piensa el sistema.

---

# Principios

Proposal Studio nunca recomienda productos.

Proposal Studio recomienda estrategias.

El seguro es una consecuencia.

No el punto de partida.

Toda recomendación debe responder tres preguntas.

- ¿Qué necesita el cliente?
- ¿Por qué?
- ¿Cómo lo resolvemos?

---

# Regla 1

## El objetivo del cliente siempre tiene prioridad

Toda propuesta comienza por el objetivo declarado.

Nunca por el producto.

El objetivo determina automáticamente:

- narrativa inicial;
- beneficios sugeridos;
- estructura del documento;
- CTA recomendado;
- categorías de alternativas;
- plantillas recomendadas;
- tono comunicacional.

El usuario podrá modificar cualquier sugerencia.

# Reglas de narrativa

Toda propuesta debe seguir el mismo recorrido argumental.

## 1.

Comprender la situación.

## 2.

Identificar riesgos.

## 3.

Explicar oportunidades.

## 4.

Presentar la estrategia.

## 5.

Comparar alternativas.

## 6.

Invitar a avanzar.

Nunca invertir este orden.

---

# Reglas de beneficios

El sistema propone beneficios automáticamente.

Los beneficios:

- pueden agregarse;
- pueden eliminarse;
- pueden editarse;
- nunca son obligatorios.

---

# Reglas de alternativas

Cada propuesta puede contener entre una y seis alternativas.

Debe existir una alternativa recomendada.

Si el usuario no selecciona una alternativa destacada, el sistema utilizará la primera.

---

# Reglas de comparativa

La comparativa debe mostrar únicamente información útil para la decisión.

Evitar columnas innecesarias.

Las filas visibles son configurables.

---

# Reglas del Resumen Ejecutivo

Toda propuesta comienza con un Resumen Ejecutivo.

Debe responder cuatro preguntas.

- ¿Qué entendimos?
- ¿Qué riesgos detectamos?
- ¿Qué estrategia proponemos?
- ¿Qué resultado esperamos?

Nunca mencionar productos antes de explicar la necesidad.

---

# Reglas del CTA

El CTA debe invitar a continuar la conversación.

Nunca presionar al cliente.

Nunca utilizar mensajes agresivos.

Siempre proponer un siguiente paso claro.

---

# Reglas de personalización

Todo contenido sugerido por Proposal Studio puede modificarse.

El asesor mantiene siempre el control.

La Inteligencia Artificial propone.

El asesor decide.

---

# Reglas de IA

La IA nunca reemplaza al asesor.

Su función consiste en:

- redactar;
- resumir;
- sugerir;
- mejorar;
- reorganizar.

Nunca tomar decisiones comerciales automáticamente.

---

# Reglas de exportación

El contenido exportado debe coincidir exactamente con la vista previa.

El PDF constituye la representación oficial de la propuesta.

---

# Reglas de consistencia

Toda modificación realizada por el usuario debe reflejarse inmediatamente en:

- vista previa;
- contenido exportado;
- versiones;
- historial.

Nunca pueden existir diferencias entre la propuesta editada y el documento generado.

---

# Reglas futuras

Toda nueva funcionalidad incorporada al producto deberá responder las siguientes preguntas.

¿Ayuda al asesor a vender mejor?

¿Hace que el cliente comprenda mejor la estrategia?

¿Reduce el tiempo necesario para construir una propuesta?

¿Mejora la calidad percibida del documento?

Si la respuesta es negativa, la funcionalidad no debería incorporarse.


# Motor de decisiones

Cuando existan múltiples reglas aplicables, Proposal Studio utilizará el siguiente orden de prioridad.

1. Decisiones explícitas del usuario.

2. Configuración de la propuesta.

3. Objetivo del cliente.

4. Tipo de cliente.

5. Reglas automáticas del sistema.

6. Valores por defecto.

Las decisiones manuales del asesor siempre prevalecen sobre cualquier automatización.
